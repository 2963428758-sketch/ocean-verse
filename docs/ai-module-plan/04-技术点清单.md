## 核心技术点（必须完成）

### 1. Spring AI 模型接入

| 维度 | 说明 |
|------|------|
| **涉及技术** | Spring AI 1.0.0 + DashScope Starter |
| **对应文件** | `AiServiceImpl.java` |
| **实现要点** | 注入 `ChatClient`，调用 `.prompt().call()` 或 `.stream()` |
| **验收标准** | 模型调用成功，返回真实回答而非 mock 数据 |
| **优先级** | P0 — 所有功能的前提 |

### 2. 多模态图像识别

| 维度 | 说明 |
|------|------|
| **涉及技术** | qwen-vl-max 视觉模型 + Spring AI 多模态 API |
| **对应文件** | `AiServiceImpl.recognizeImage()` |
| **实现要点** | 构造 `UserMessage` 携带图片 URL + 文本提示，解析 JSON 结构化输出 |
| **验收标准** | 上传海洋生物图片，返回正确的物种名称和置信度 |
| **优先级** | P0 |

### 3. SSE 流式对话

| 维度 | 说明 |
|------|------|
| **涉及技术** | Server-Sent Events + ChatClient.stream() + Reactor |
| **对应文件** | `AiServiceImpl.chatStream()` |
| **实现要点** | `.stream().content()` 返回 `Flux<String>`，每个 chunk 写入 `HttpServletResponse` |
| **验收标准** | 首 Token 延迟 < 1s，流式输出流畅，结束时发送 `[DONE]` |
| **优先级** | P0 |

### 4. 配置属性绑定

| 维度 | 说明 |
|------|------|
| **涉及技术** | `@ConfigurationProperties` |
| **对应文件** | `AiProperties.java`（新建） |
| **实现要点** | 绑定 `oceanverse.ai.*` 的 image-model / chat-model / embedding-model |
| **验收标准** | 修改 YAML 配置后，重启应用能反映到 Java 实例中 |
| **优先级** | P0 |

### 5. Redis Stream 事件发布

| 维度 | 说明 |
|------|------|
| **涉及技术** | `RedisTemplate.opsForStream()` + `StreamRecords` |
| **对应文件** | `AiServiceImpl.java`（新增辅助方法） |
| **实现要点** | 识别完成和问答完成后，向 `stream:ai` 发送 `AiEvent` 消息 |
| **验收标准** | `AiStreamConsumer` 能收到事件并触发通知推送 |
| **优先级** | P1 |

## 进阶技术点（推荐完成）

### 6. Prompt 工程

| 维度 | 说明 |
|------|------|
| **涉及技术** | System Prompt 设计 + Few-shot + 结构化输出约束 |
| **对应文件** | `PromptTemplateManager.java`（新建） |
| **实现要点** | 为 SPECIES / ECOLOGY / CONSERVATION / GENERAL 四种类型设计差异化 Prompt |
| **验收标准** | 不同类型问题的回答风格和内容结构有明显差异 |
| **优先级** | P1 |

### 7. RAG 检索增强

| 维度 | 说明 |
|------|------|
| **涉及技术** | text-embedding-v3 + Redis Stack 向量检索 + Spring AI VectorStore |
| **对应文件** | `KnowledgeBaseService.java` + `DocumentLoader.java` + `RedisVectorStoreConfig.java`（新建） |
| **实现要点** | 从 species 表加载文档 → 向量化存入 Redis → 问答时检索 Top-K 相关片段 → 注入 System Prompt |
| **验收标准** | 回答中引用的信息与数据库一致，无幻觉内容；检索不到时降级为裸模型 |
| **优先级** | P1 |

### 8. 多轮对话上下文管理

| 维度 | 说明 |
|------|------|
| **涉及技术** | Redis List + Spring AI Message 抽象 + 上下文窗口裁剪 |
| **对应文件** | `ChatSessionManager.java`（新建） |
| **实现要点** | 按 sessionId 缓存最近 N 轮对话，每次请求携带历史消息列表 |
| **验收标准** | 连续提问时 AI 能理解上下文指代（如"它"指上一轮提到的物种） |
| **优先级** | P1 |

### 9. 语义缓存

| 维度 | 说明 |
|------|------|
| **涉及技术** | Embedding 向量 + Redis 缓存 + MD5 哈希（简化版） |
| **对应文件** | `SemanticCacheService.java`（新建） |
| **实现要点** | 新问题先检索缓存，命中则直接返回；高质量回答自动缓存 |
| **验收标准** | 相同问题第二次提问响应 < 500ms；缓存命中率 > 20% |
| **优先级** | P2 |

### 10. API 调用限流

| 维度 | 说明 |
|------|------|
| **涉及技术** | Redis INCR + 滑动窗口 + 用户等级 |
| **对应文件** | `AiRateLimiter.java`（新建） |
| **实现要点** | 普通用户每日 50 次问答 / 10 次识别，超限返回友好提示 |
| **验收标准** | 第 51 次问答被拦截；每日 0 点自动重置；`GET /api/ai/quota` 返回剩余次数 |
| **优先级** | P2 |

## 拓展技术点（锦上添花）

### 11. AI 效果评估

| 维度 | 说明 |
|------|------|
| **涉及技术** | MyBatis-Plus 聚合查询 + 统计指标计算 |
| **对应文件** | `AiEvalService.java`（新建） |
| **实现要点** | 满意度分布、平均响应时间、热门问题 Top 10、识别准确率 |
| **验收标准** | `GET /api/ai/eval/stats` 返回完整统计数据 |
| **优先级** | P2 |

### 12. 跨模块联动

| 维度 | 说明 |
|------|------|
| **涉及技术** | 跨模块服务调用 + 数据聚合 |
| **对应文件** | `AiServiceImpl.java`（修改） |
| **实现要点** | 识别结果关联 species 表；高置信度时建议创建生态观测记录 |
| **验收标准** | 识别返回物种卡片；点击"记录观测"能创建 eco_observation 记录 |
| **优先级** | P3 |

### 13. 社区验证机制

| 维度 | 说明 |
|------|------|
| **涉及技术** | 众包审核 + 置信度阈值 |
| **对应文件** | 利用 `image_recognition` 表已有的 `verification_status` 字段 |
| **实现要点** | 高置信度自动 VERIFIED，低置信度进入社区审核队列 |
| **验收标准** | 低置信度识别记录在前端展示"待审核"标记，社区用户可投票验证 |
| **优先级** | P3 |

### 14. 个性化推荐问题

| 维度 | 说明 |
|------|------|
| **涉及技术** | 用户行为分析 + 协同过滤（简化版） |
| **对应文件** | `Chat.vue`（修改） |
| **实现要点** | 根据用户浏览过的物种、参与过的讨论，生成个性化推荐问题列表 |
| **验收标准** | 不同用户看到的推荐问题不同；推荐内容与用户兴趣相关 |
| **优先级** | P3 |

### 15. 物种知识卡片

| 维度 | 说明 |
|------|------|
| **涉及技术** | 知识聚合 + 前端组件渲染 |
| **对应文件** | `Recognize.vue`（修改） |
| **实现要点** | 识别后展示完整物种信息：分布地图、保护等级、生态角色、趣味知识 |
| **验收标准** | 识别结果页渲染出物种卡片，包含图片、分布、保护等级等信息 |
| **优先级** | P3 |

## 验收检查清单

完成所有技术点后，逐项检查：

- [ ] 图像识别能返回真实物种名称和置信度
- [ ] 智能问答首 Token 延迟 < 1s，回答专业准确
- [ ] 多轮对话能理解上下文指代
- [ ] RAG 检索命中时，回答内容与数据库一致
- [ ] 语义缓存命中时，响应时间 < 500ms
- [ ] 超过调用限制时，返回友好提示
- [ ] 识别完成后，用户收到通知推送
- [ ] 高置信度识别后，能一键创建观测记录
- [ ] AI 效果评估接口返回完整统计数据
- [ ] 所有异常情况（模型不可用、超时、解析失败）都有优雅降级
