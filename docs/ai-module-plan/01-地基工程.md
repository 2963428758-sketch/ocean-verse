## 任务 1.2：接入真实图像识别模型

### 背景

当前 `AiServiceImpl.recognizeImage()` 返回硬编码的"待识别"结果：

```java
// 当前 mock 实现
record.setPredictedSpeciesName("待识别");
record.setConfidenceScore(new java.math.BigDecimal("0.00"));
record.setAiModelVersion("mock-v1.0");
```

需要替换为调用 DashScope 的 `qwen-vl-max` 多模态模型。

### 实施步骤

**修改文件**：`smart-ai-service/src/main/java/com/oceanverse/ai/service/impl/AiServiceImpl.java`

**第一步：注入依赖**

**第二步：实现真实识别逻辑**

```java
@Override
public Object recognizeImage(MultipartFile file, Double latitude, Double longitude) {
    long startTime = System.currentTimeMillis();
    String imageUrl = ossUtil.upload(file);
    
    ImageRecognition record = new ImageRecognition();
    record.setRecognitionCode("REC" + UUID.randomUUID().toString().substring(0, 8));
    record.setImageUrl(imageUrl);
    record.setFileName(file.getOriginalFilename());
    record.setFileSize(file.getSize());
    record.setRecognitionType("AUTO");
    record.setVerificationStatus("PENDING");
    record.setCreateTime(LocalDateTime.now());
    record.setUpdateTime(LocalDateTime.now());

    try {
        // 构造多模态请求：图片 + 提示词
        String prompt = """
            请识别这张图片中的海洋生物，返回以下 JSON 格式：
            {
              "speciesName": "中文物种名",
              "scientificName": "拉丁学名",
              "confidence": 0.95,
              "description": "简要描述（50字以内）",
              "habitat": "主要栖息地",
              "conservationStatus": "保护等级"
            }
            只返回 JSON，不要其他文字。
            """;

        // 调用 qwen-vl-max 视觉模型
        String response = chatClient.prompt()
                .user(u -> u
                    .text(prompt)
                    .media(MimeTypeUtils.IMAGE_JPEG, new URL(imageUrl))
                )
                .call()
                .content();

        // 解析模型返回的 JSON
        JsonNode result = objectMapper.readTree(response);
        
        record.setPredictedSpeciesName(result.get("speciesName").asText());
        record.setConfidenceScore(new BigDecimal(result.get("confidence").asText()));
        record.setRecognitionResult(response);  // 保存完整 JSON
        record.setAiModelVersion(aiProperties.getImageModel());
        record.setVerificationStatus(
            result.get("confidence").asDouble() > 0.8 ? "VERIFIED" : "PENDING"
        );

    } catch (Exception e) {
        log.error("图像识别失败: {}", e.getMessage(), e);
        record.setAiModelVersion(aiProperties.getImageModel());
        record.setPredictedSpeciesName("识别失败");
        record.setErrorMessage(e.getMessage());
    }

    record.setProcessingTimeMs((int)(System.currentTimeMillis() - startTime));
    recognitionMapper.insert(record);

    // 发布识别完成事件到 Redis Stream
    publishRecognitionEvent(record);

    return record;
}
```

### 关键技术点

1. **多模态输入**：`qwen-vl-max` 支持图片 URL + 文本提示的组合输入
2. **结构化输出**：通过 Prompt 约束模型返回 JSON 格式，便于程序解析
3. **置信度阈值**：> 0.8 自动标记为 VERIFIED，否则进入人工审核队列
4. **异常降级**：模型调用失败时记录错误信息，但不阻断流程

### 验收标准

- [ ] 上传一张海洋生物图片，能返回正确的物种名称和置信度
- [ ] 置信度 > 0.8 的记录 `verification_status` 为 VERIFIED
- [ ] 识别失败的记录保存了 `error_message`，不会抛 500 错误
- [ ] `processing_time_ms` 字段记录了实际耗时

## 任务 1.3：接入真实智能问答模型

### 背景

当前 `AiServiceImpl.chatStream()` 返回固定文案的 mock 流式响应：

```java
// 当前 mock 实现
String mockAnswer = "感谢您的提问！这是一个关于「" + dto.getQuestion() + "」的好问题。" +
        "目前 AI 服务为演示模式...";
```

需要替换为调用 DashScope 的 `qwen-plus` 流式对话 API。

### 实施步骤

**修改文件**：同 `AiServiceImpl.java`

```java
@Override
public void chatStream(ChatDTO dto, HttpServletResponse response) {
    long startTime = System.currentTimeMillis();
    
    try {
        PrintWriter writer = response.getWriter();
        
        // 保存问答记录（先创建，后续更新回答内容）
        QaHistory history = new QaHistory();
        history.setQuestionCode("QA" + UUID.randomUUID().toString().substring(0, 8));
        history.setQuestionType(dto.getQuestionType() != null ? dto.getQuestionType() : "GENERAL");
        history.setQuestionText(dto.getQuestion());
        history.setSessionId(dto.getSessionId());
        history.setAiModelVersion(aiProperties.getChatModel());
        history.setCreateTime(LocalDateTime.now());
        history.setUpdateTime(LocalDateTime.now());
        history.setStatus(1);

        // 构造系统提示词（海洋专家角色）
        String systemPrompt = """
            你是一位资深的海洋生物学家，拥有 20 年的海洋生态研究经验。
            你的专长包括：海洋生物分类学、珊瑚礁生态、海洋保护、深海生物学。
            回答要求：
            1. 专业准确，引用可靠知识
            2. 语言通俗易懂，适合大众理解
            3. 如果不确定，明确告知"这个问题我暂时没有确切答案"
            4. 回答控制在 300 字以内
            """;

        // 使用 Spring AI 的流式 API
        StringBuilder fullAnswer = new StringBuilder();
        
        chatClient.prompt()
            .system(systemPrompt)
            .user(dto.getQuestion())
            .stream()
            .content()
            .doOnNext(chunk -> {
                try {
                    fullAnswer.append(chunk);
                    writer.write("data: " + chunk + "\n\n");
                    writer.flush();
                } catch (Exception e) {
                    log.error("写入 SSE 流失败", e);
                }
            })
            .doOnComplete(() -> {
                try {
                    writer.write("data: [DONE]\n\n");
                    writer.flush();
                    
                    // 更新完整回答和耗时
                    history.setAnswerText(fullAnswer.toString());
                    history.setProcessingTimeMs((int)(System.currentTimeMillis() - startTime));
                    qaHistoryMapper.insert(history);
                    
                    // 发布问答完成事件
                    publishChatEvent(history);
                } catch (Exception e) {
                    log.error("完成 SSE 流失败", e);
                }
            })
            .doOnError(e -> {
                log.error("AI 流式调用失败", e);
                try {
                    writer.write("data: [ERROR] AI 服务暂时不可用，请稍后重试\n\n");
                    writer.flush();
                } catch (Exception ex) {
                    log.error("写入错误响应失败", ex);
                }
            })
            .subscribe();

    } catch (Exception e) {
        log.error("SSE 流式响应异常: ", e);
    }
}
```

### 关键技术点

1. **流式响应**：`chatClient.prompt().stream()` 返回 `Flux<String>`，每个 chunk 实时写入 SSE
2. **系统角色设定**：通过 `.system()` 设置海洋专家角色，约束回答风格和专业度
3. **响应式编程**：使用 Reactor 的 `doOnNext` / `doOnComplete` / `doOnError` 处理流生命周期
4. **异步保存**：在流完成后才保存完整回答到数据库，避免阻塞流式输出

### 验收标准

- [ ] 提问后能在 1 秒内开始收到流式响应（首 Token 延迟 < 1s）
- [ ] 回答内容与海洋生物相关，专业且准确
- [ ] 流结束后 `qa_history` 表中有完整记录（含 `answer_text` 和 `processing_time_ms`）
- [ ] AI 服务不可用时返回友好错误提示，不抛 500

## 任务 1.4：实现 Redis Stream 事件发布

### 背景

`message-service` 中的 `AiStreamConsumer` 已就绪，监听 `stream:ai`，但 `AiServiceImpl` 从未发布事件。

### 实施步骤

**在 AiServiceImpl 中添加辅助方法**：

```java
/**
 * 发布图像识别完成事件
 */
private void publishRecognitionEvent(ImageRecognition record) {
    try {
        AiEvent event = AiEvent.builder()
                .action(CommonConstants.ACTION_RECOGNITION_COMPLETE)
                .userId(record.getUserId())
                .recognitionCode(record.getRecognitionCode())
                .speciesName(record.getPredictedSpeciesName())
                .confidence(record.getConfidenceScore() != null 
                    ? record.getConfidenceScore().doubleValue() : 0.0)
                .build();
        
        Map<String, String> payload = new HashMap<>();
        payload.put("action", event.getAction());
        payload.put("payload", objectMapper.writeValueAsString(event));
        
        redisTemplate.opsForStream().add(
            StreamRecords.string(payload).withStreamKey(CommonConstants.STREAM_AI)
        );
        
        log.info("已发布识别完成事件: {}", record.getRecognitionCode());
    } catch (Exception e) {
        log.error("发布识别事件失败", e);
        // 事件发布失败不影响主流程
    }
}

/**
 * 发布智能问答完成事件
 */
private void publishChatEvent(QaHistory history) {
    try {
        AiEvent event = AiEvent.builder()
                .action(CommonConstants.ACTION_CHAT_SESSION)
                .userId(history.getUserId())
                .questionType(history.getQuestionType())
                .build();
        
        Map<String, String> payload = new HashMap<>();
        payload.put("action", event.getAction());
        payload.put("payload", objectMapper.writeValueAsString(event));
        
        redisTemplate.opsForStream().add(
            StreamRecords.string(payload).withStreamKey(CommonConstants.STREAM_AI)
        );
        
        log.info("已发布问答完成事件: {}", history.getQuestionCode());
    } catch (Exception e) {
        log.error("发布问答事件失败", e);
    }
}
```

### 关键技术点

1. **事件解耦**：AI 服务只负责发布事件，通知推送由 `AiStreamConsumer` 处理
2. **容错设计**：事件发布失败不影响主流程（try-catch 包裹）
3. **StreamRecords**：使用 `StreamRecords.string()` 构造 Redis Stream 消息

### 验收标准

- [ ] 图像识别完成后，用户收到"图像识别已完成"的通知推送
- [ ] 智能问答完成后，`AiStreamConsumer` 日志中能看到 `CHAT_SESSION` 事件
- [ ] 事件发布失败不会导致 AI 接口报错

## 任务 1.5：更新问答反馈接口

### 背景

当前 `feedback()` 方法功能完整，但可以增强：记录反馈事件到 Redis Stream。

### 实施步骤

```java
@Override
public void feedback(Long id, Integer feedback) {
    QaHistory history = qaHistoryMapper.selectById(id);
    if (history != null) {
        history.setFeedback(feedback);
        history.setFeedbackText(feedback == 1 ? "满意" : "不满意");
        history.setUpdateTime(LocalDateTime.now());
        qaHistoryMapper.updateById(history);
        
        // 发布反馈事件
        try {
            AiEvent event = AiEvent.builder()
                    .action(CommonConstants.ACTION_FEEDBACK)
                    .userId(history.getUserId())
                    .questionType(history.getQuestionType())
                    .build();
            
            Map<String, String> payload = new HashMap<>();
            payload.put("action", event.getAction());
            payload.put("payload", objectMapper.writeValueAsString(event));
            
            redisTemplate.opsForStream().add(
                StreamRecords.string(payload).withStreamKey(CommonConstants.STREAM_AI)
            );
        } catch (Exception e) {
            log.error("发布反馈事件失败", e);
        }
    }
}
```

### 验收标准

- [ ] 用户点击"满意/不满意"后，数据库正确更新
- [ ] `AiStreamConsumer` 能收到 `FEEDBACK` 事件
