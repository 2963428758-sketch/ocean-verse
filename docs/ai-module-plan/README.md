# OceanVerse AI 模块 — 实施任务手册

> **目标**：将 AI 模块从 mock 骨架升级为具备真实模型调用、RAG 检索增强、多轮对话、语义缓存等能力的智能服务层。

## 项目背景

OceanVerse 是一个智慧海洋探索平台，AI 模块（`smart-ai-service`）承载图像识别和智能问答两大核心功能。当前模块的**基础设施已完整**（数据库、API、前端、依赖配置、Redis Stream 消费者全部就绪），但核心服务逻辑 `AiServiceImpl` 仍为 mock 实现。

## 快速开始

如果你是第一次接触这个模块，建议按以下顺序阅读：

1. **先看现状**：阅读 `smart-ai-service/src/main/java/com/oceanverse/ai/service/impl/AiServiceImpl.java`，理解当前 mock 实现
2. **再看配置**：阅读 `oceanverse-app/src/main/resources/application.yml` 中的 `spring.ai` 和 `oceanverse.ai` 配置段
3. **然后看管道**：阅读 `message-service` 中的 `AiStreamConsumer.java`，理解事件消费者已就绪
4. **最后看任务**：按 [01-地基工程.md](./01-地基工程.md) 开始实施

## 关键约束

1. **API Key 管理**：DashScope API Key 通过环境变量 `DASHSCOPE_API_KEY` 或 `application-local.yml` 注入，**绝不能硬编码或提交到 Git**
2. **成本控制**：qwen-plus 按 Token 计费，必须实现缓存和限流机制
3. **响应时间**：图像识别目标 < 5s，问答首 Token 延迟目标 < 1s
4. **错误降级**：AI 服务不可用时，返回友好提示而非 500 错误
